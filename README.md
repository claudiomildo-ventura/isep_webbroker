# ISEP | Integrated Software Engineering Platform - WebBroker

---

## 🧩 Project Overview

This application is a modernization of a legacy system originally developed in Object Pascal (Delphi XE10), now structured as a WebBroker-based API service. The project aims to provide a robust, maintainable, and scalable backend solution, leveraging the strengths of Delphi for web server development.

## 🛠 Technologies Used

The project utilizes the following technologies:

- **Delphi XE10 (Object Pascal)**: Core language and framework for application logic and web server functionality.
- **WebBroker**: Framework for building web server applications in Delphi.
- **HTML, CSS, JavaScript**: For web interface and static content delivery.
- **RESTful API**: Exposes endpoints for integration with other systems.
- **Windows**: Target operating system for deployment.
- **Git & GitHub**: Version control and collaboration.

## 📁 Project Structure

- `isep_webbroker.dpr` / `.dproj`: Delphi project files.
- `ServerContainer.pas` / `.dfm`: Server container logic and design.
- `ServerController.pas`: Implementation of server-side methods and API endpoints.
- `WebModule.pas` / `.dfm`: Web module configuration and routing.
- `Startup.pas` / `.dfm`: Application startup logic.
- `css/`, `js/`, `images/`, `templates/`: Static assets and HTML templates.

## 🚀 Getting Started

1. **Requirements**:
   - Delphi XE10 or compatible IDE
   - Windows OS

2. **Build & Run**:
   - Open the `.dpr` project in Delphi.
   - Build and run the application.
   - Access the API and web interface via the configured port.

3. **Development**:
   - Modify `.pas` and `.dfm` files for business logic and UI changes.
   - Update static assets in `css/`, `js/`, and `templates/` as needed.
